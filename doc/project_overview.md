# FlowZint V2 — Complete Architecture & Project Overview

## 1. Project Overview
FlowZint is a multi-tenant, AI-powered sales automation platform. It is designed to act as an automated sales representative that can make outbound calls to leads, qualify them, detect buying intent, handle objections, and book demos—all without human intervention. The system also includes a sophisticated post-call analysis pipeline to score leads, extract insights, and iteratively improve the AI agent's performance.

## 2. Core Architecture Components
The project is structured as a monorepo containing three primary systems that work together:

### 2.1 Backend (Node.js + Express + TypeScript)
The backend acts as the central orchestrator of the entire platform. 
* **Database**: PostgreSQL with Prisma ORM.
* **Responsibilities**: 
  * Manages authentication (JWT, Roles) and multi-tenant isolation via **Workspaces**.
  * Handles Business Onboarding: Companies upload documents (PDF/DOCX) and define their target customers and call objectives.
  * Generates and versions AI personas (`AgentConfig`) based on the business context.
  * Manages Leads, Bookings (via Calendly API stub), and Integrations (CRM sync).
  * Exposes APIs for the frontend dashboard and internal APIs for the Python AI agent.
  * Runs the **Post-Call Analysis Pipeline**: Webhooks from LiveKit trigger OpenAI (GPT-4o) to analyze call transcripts (sentiment, scoring, intent, objections).
  * Runs the **Learning Loop Engine**: Analyzes batches of calls to identify common objections and automatically iterates the agent's script.

### 2.2 AI Voice Agent (Python + LiveKit)
This is the real-time telephony engine responsible for actually speaking to leads.
* **Technology**: Python, LiveKit, OpenAI Realtime API (`gpt-realtime-2025-08-28`).
* **Responsibilities**:
  * Outbound caller agent (`sales_caller.py`) designed to dynamically adopt the persona configured in the backend's `AgentConfig`.
  * Converses with leads with sub-500ms latency.
  * Utilizes function calling (tools) during the conversation:
    * `qualify_lead`: Saves lead qualification status.
    * `book_demo`: Triggers backend integration to schedule a meeting.
    * `transfer_to_human`: Hands off the call if needed.
    * `detected_answering_machine`: Leaves a voicemail and hangs up.

### 2.3 Frontend Dashboard (Next.js 14)
The user-facing CRM and management portal.
* **Technology**: Next.js 14, Tailwind CSS, Shadcn/ui, Recharts.
* **Responsibilities**:
  * **Onboarding Wizard**: Allows businesses to create a workspace, input context, and upload knowledge documents.
  * **Dashboard Stats**: Displays call volume, success rates, top objections, and average lead scores.
  * **Lead Management**: Displays a list of leads (Hot/Warm/Cold), with details and timelines.
  * **Call History & Analysis**: A detailed view of past calls featuring an audio player, a scrollable transcript, and AI-generated analysis (sentiment rings, objection chips).
  * **Agent Configuration**: An interface to view and tweak the AI's prompt and scripts, including reviewing AI-suggested "Learning" iterations.

## 3. Database Schema Overview
The database uses a multi-tenant model where almost all entities belong to a `Workspace`:
* **Users & Workspaces**: Users can own or be members of Workspaces.
* **BusinessContext & BusinessDocument**: Stores the knowledge base and identity of the business using the workspace.
* **AgentConfig**: Versions of the AI prompt and objection handlers.
* **Lead**: The people being called.
* **Call & CallAnalysis**: Stores the status of the SIP call and the resulting AI analysis (sentiment, score, buying intent).
* **LearningInsight**: Actionable feedback generated by analyzing multiple past calls to improve the `AgentConfig`.
* **Booking**: Scheduled meetings.

## 4. How It All Works Together (The Lifecycle)

### Step 1: Business Onboarding
1. A user registers and creates a **Workspace**.
2. They fill out their **Business Context** (product description, pricing, competitors) and upload PDFs/Documents.
3. The backend extracts text from the documents and uses GPT-4o to generate an initial **AgentConfig** (system prompt, opening script, objection handlers).

### Step 2: The Outbound Call
1. The user uploads **Leads** via the frontend dashboard.
2. The user initiates a call to a lead.
3. The backend triggers a dispatch to the **Python AI Agent** via LiveKit.
4. The Python agent fetches the active `AgentConfig` for that workspace and dials the lead via SIP (Twilio).
5. The AI converses with the lead in real-time. If the lead is interested, the AI triggers the `book_demo` tool.

### Step 3: Post-Call Analysis
1. Once the call ends, LiveKit sends a webhook back to the Node.js backend.
2. The backend fetches the transcript and runs it through an OpenAI structured-output prompt.
3. A `CallAnalysis` record is created with a score (1-100), sentiment, and extracted objections.
4. An email summary is optionally sent to the sales rep via Resend.

### Step 4: The Learning Loop
1. Over time, the backend analyzes batches of completed calls.
2. If it notices a recurring pattern (e.g., "leads always ask about a specific competitor"), it generates a `LearningInsight`.
3. The backend drafts a **new version** of the `AgentConfig` that includes a new objection handler to specifically address this gap.
4. The user reviews the updated config in the dashboard and activates it, completing the feedback loop and creating a smarter AI sales rep.

## 5. Backend API Routes Summary

### Authentication
* `POST /api/auth/register` — Register a new user
* `POST /api/auth/login` — Login and receive JWT
* `POST /api/auth/refresh` — Refresh JWT token

### Leads
* `GET    /api/leads` — List leads (paginated, filtered)
* `POST   /api/leads` — Create a new lead
* `GET    /api/leads/:id` — Get lead details
* `PUT    /api/leads/:id` — Update lead
* `DELETE /api/leads/:id` — Delete lead
* `POST   /api/leads/bulk` — Import leads via CSV

### Calls
* `GET    /api/calls` — Call history
* `POST   /api/calls/dispatch` — Trigger an outbound call via LiveKit
* `GET    /api/calls/:id` — Call details + transcript
* `POST   /api/calls/webhook` — LiveKit webhook handler (triggered when call ends)

### Analysis
* `GET    /api/analysis/:callId` — Get AI analysis for a specific call
* `POST   /api/analysis/trigger` — Manually re-run analysis

### Dashboard
* `GET    /api/dashboard/stats` — Aggregate stats (e.g., total calls, success rate)
* `GET    /api/dashboard/charts` — Chart data (volume, scores, objections)

### Bookings
* `POST   /api/bookings/create` — Create Calendly booking + send confirmation email
* `GET    /api/bookings` — Booking list

## 6. Frontend Routes Summary (Next.js App Router)

### Public Routes
* `/` — Landing/Home page

### Dashboard (Protected Routes)
* `/dashboard` — Main dashboard (Stats, charts)

* `/dashboard/learning` — Review AI-suggested iterations and learning loop insights
